# 5-Meta - Vault Management & Navigation

## Purpose

Meta contains the **structural elements** that make the knowledge base work: templates, maps of content (MOCs), guidelines, and vault management documentation.

**Key Principle:** Meta notes are about the vault itself, not the content within it.

## Templates

Templates ensure **consistency and quality** across the vault.

**Available Templates:**
- **project-template.md** - For time-bound projects
- **area-template.md** - For ongoing responsibilities
- **bookmark-template.md** - For web bookmarks
- **video-template.md** - For video resources
- **article-template.md** - For articles and blog posts
- **permanent-note-template.md** - For evergreen knowledge notes

**Using Templates:**
1. In Obsidian: `Ctrl/Cmd + P` → "Templates: Insert template"
2. Choose appropriate template
3. Fill in placeholders
4. Save and process

See [[5-Meta/Templates/README]] for details on each template.

## Maps of Content (MOCs)

MOCs are **navigation hubs** that organize related notes by topic.

**Why MOCs?**
- Provide high-level topic overviews
- Enable non-linear navigation
- Show relationships between concepts
- Support learning progressions
- Reduce reliance on folders

**Current MOCs:**
- [[5-Meta/MOCs/AI-Models-MOC]] - All AI models, organized by category
- [[5-Meta/MOCs/Coding-Assistants-MOC]] - Coding tools and comparisons
- [[5-Meta/MOCs/Prompting-Techniques-MOC]] - Prompt engineering patterns
- [[5-Meta/MOCs/Learning-Path-MOC]] - Structured learning progressions

**Creating a MOC:**
- Focus on a specific topic or theme
- Link to all related notes
- Provide context and organization
- Update as content grows
- Keep it navigable (not exhaustive)

## Vault Maintenance

**Weekly:**
- Process Inbox using [[5-Meta/Guidelines/Processing-Workflow]]
- Update relevant MOCs with new content
- Check for broken links

**Monthly:**
- Review and update time-sensitive content
- Update templates if needed
- Review lessons learned

**Quarterly:**
- Major MOC updates
- Template improvements
- Guidelines refinement
- Structure adjustments
